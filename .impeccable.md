## Design Context

### Users
- **Primary:** Recruiters and hiring managers evaluating Kyle as a candidate — scanning quickly on laptops during work hours, looking for signal of competence and professionalism.
- **Secondary:** Fellow software engineers — browsing with a more technical eye, noticing craft details, code quality, and aesthetic choices.
- **Future:** Potential climbing section for the broader outdoor/climbing community — people who value authenticity and physicality.

### Brand Personality
**Bold. Retro. Crafted.**

The interface should feel like something built by hand with intention — not generated, not templated. It communicates that Kyle cares about the details, has taste beyond the default, and approaches engineering the same way: deliberately.

### Emotional Goal
Visitors should walk away thinking: "This person has technical chops *and* taste." The portfolio should feel like an artifact someone made, not a form someone filled out.

### Aesthetic Direction
**Dual-personality theme — both modes are intentional and staying:**

- **Light mode — Neo-brutalist paper/craft:** Warm cream backgrounds, hard-edge box shadows (no blur), heavy 2px borders in dark olive-brown, sharp rectangles with no rounded corners. Feels like a zine or a hand-printed broadsheet. Sans-serif body text grounds it in utility.
- **Dark mode — CRT amber terminal:** Black background with amber/gold monochrome palette, monospace typography, scanline overlay, vignette, subtle flicker. Full retro-computing atmosphere. Feels like accessing someone's personal mainframe.

**Known issue:** Light mode on the resume page feels undercooked compared to the rest of the site. Needs the same level of craft and intentionality as the other pages.

**Theme choice rationale:** Light mode for the daytime professional scanning context (recruiters). Dark mode as the personality play that fellow engineers will appreciate and remember.

### Design Principles
1. **Every element is a decision.** No defaults. If it's on the page, it was chosen — the font, the shadow, the spacing, the color. Nothing should look like it was left at framework defaults.
2. **Two modes, one identity.** Light and dark aren't just color swaps. They're two expressions of the same personality: deliberate, slightly irreverent, built with care.
3. **Show the craft.** The interface itself is a portfolio piece. Hard-edge shadows, physical button presses, scroll reveals — these aren't decoration, they're demonstrations of engineering taste.
4. **Substance over spectacle.** Bold choices yes, but in service of readability and usability. A recruiter should find what they need in 10 seconds. An engineer should want to linger.
5. **Leave room to grow.** The architecture should accommodate a climbing section and other personal dimensions without feeling bolted on.
